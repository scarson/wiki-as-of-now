# Eligibility gold set — provenance

`eligibility-set.json` is the frozen gold set for the safe-lane eligibility gate
(biographies of living persons / BLP exclusion + namespace + freshness + advisory
signals). It is consumed by `test/safelane/eligibility-gold.test.ts`, which maps each
raw envelope through the **real ingest mapper** (`mapResponseToMetadata` →
`toArticleMetadata` in `src/ingest/wikimedia.ts`) and then the **real gate**
(`evaluateEligibility` in `src/safelane/eligibility.ts`), so the probe/normalization
and advisory-scan paths are actually exercised — not pre-cleaned fields.

## Shape

```jsonc
{
  "goldNow": "2026-06-07T00:00:00Z",   // single injected `now`; ≥15 min after every real revision
  "entries": [
    { "name": "...", "rawResponse": { /* FULL Action-API body */ }, "expected": { "eligibility": "...", "reasons": [...] } }
  ]
}
```

`expected` is **hand-derived from spec §2** (the check table + canonical reason order),
NOT generated by running the gate — otherwise the test would be tautological
(testing-pitfalls §9, gold-set honesty). The test re-derives the verdict with the real
gate and asserts it equals the hand-written `expected`, so a gate regression fails loudly.

## Capture

- **Date:** 2026-06-06. **Endpoint:** `https://en.wikipedia.org/w/api.php`.
- **Request shape:** identical to what `fetchArticle` builds —
  `action=query&prop=revisions|categories|info&rvprop=content|ids|timestamp&rvslots=main`,
  `clcategories=<canonical BLP-set, pipe-joined>`, `formatversion=2`, `maxlag=5`,
  `redirects=1`, descriptive User-Agent (G14).
- **Frozen:** the FULL response body is stored verbatim per entry; tests never re-fetch
  (the suite stays network-free and deterministic). Re-capture only on the compliance
  doc's review cadence, and re-derive `expected` by hand if you do.
